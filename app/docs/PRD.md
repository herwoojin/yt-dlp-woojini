# PRD — yt-dlp web app

> Product Requirements Document
> 마지막 갱신: 2026-05-20

## 1. 한 줄 요약

YouTube URL 하나로 **영상 다운로드 + 시계열 목차 + 짧은 요약 + 이메일/블로그용 HTML**을 자동 생성해서 **로컬에 저장**하고, 모바일에서도 어디서든 접근할 수 있는 개인용 웹앱.

## 2. 목적과 배경

### 문제
- 유튜브 영상을 정리해 블로그/카페에 올리고 싶으면 매번 직접 시청, 받아쓰기, 요약, HTML 정리를 반복.
- 영상 본체도 보관하고 싶지만 일일이 yt-dlp 명령 외워서 쓰기 번거롭다.
- 모바일에서 발견한 영상은 일단 텔레그램으로 보내두고 나중에 PC에서 처리 = 흐름이 끊김.

### 해결
- 휴대폰/PC 어디서든 웹페이지 또는 텔레그램으로 URL 한 줄 보내면 끝.
- 영상은 본인 맥에 그대로 저장 (저작권/스토리지/지속성 보장).
- 4가지 출력물 자동 생성:
  1. **transcript.txt** — 가장 작은 원본 스크립트
  2. **chapters.json** — 시계열 목차 (Gemini가 영상 흐름 이해 후 생성)
  3. **email_readable.html** — 이메일 본문에 붙여넣기 좋은 가독성 HTML
  4. **blog_long.html** — 블로그/카페용 장문 HTML
  5. **summary_short.html** — 3~5문장 짧은 요약

## 3. 타깃 사용자

| 페르소나 | 사용 시나리오 |
|----------|--------------|
| **블로거/카페 운영자** | 유튜브 강의 시청 후 정리글 작성. 영상 한 번 등록하면 블로그 초안 HTML이 나옴 |
| **콘텐츠 큐레이터** | 매일 영상 5~10개 보고 짧은 요약을 SNS/이메일로 공유 |
| **자료 보관 매니아** | 사라질 수도 있는 영상을 로컬에 보관 + 검색 가능한 텍스트로 추출 |
| **모바일 헤비 유저** | PC 안 켜고 휴대폰만으로 영상 등록 → 나중에 PC에서 결과 확인 |

## 4. 핵심 기능 (MVP)

### F1. URL 입력 → 잡 등록
- 입력: YouTube URL + Gemini 품질(Flash/Pro)
- 진입점: 웹페이지 폼, 텔레그램 봇, REST API

### F2. 비동기 잡 처리
- 단일 워커가 순차 처리 (디스크/쿼터 보호)
- 단계: 다운로드 → 자막 추출 → transcript 생성 → Gemini 4종 생성
- 각 단계 실패 시 graceful fallback (잡 자체는 가능한 한 완료로)

### F3. 로컬 디스크 저장
- 기본 경로: `~/yt-dlp-downloads/<job_id>/`
- 설정에서 경로 변경 가능 (UI에서 즉시 반영)
- 모든 산출물 보존 (job.json 메타 포함)

### F4. 모바일 반응형 UI
- iPhone/Android 브라우저에서 한 손 조작 가능
- 잡 목록, 진행상황, 산출물 다운로드, HTML 미리보기

### F5. 텔레그램 봇 통합
- BotFather 토큰만 넣으면 자동 polling 시작
- 50MB 이하 영상은 텔레그램으로도 회신
- 처리 완료 시 푸시 알림

### F6. 외부 접근
- cloudflared quick tunnel (즉시) 또는 Tailscale Funnel (영구 URL)
- 어디서든 휴대폰으로 접속

### F7. Firebase Google 로그인
- 본인만 자기 백엔드 호출 가능 (ID 토큰 검증)
- 개발 모드에선 `ALLOW_INSECURE_AUTH=true`로 우회

### F8. 도움말 패널 (앱 내)
- 6단계 가이드 (Gemini 키, 텔레그램, 백엔드 재시작 등)
- 명령은 클릭으로 복사

## 5. 비기능 요구사항

| 항목 | 목표 |
|------|------|
| **응답성** | 모바일에서 페이지 로드 1초 이내 |
| **처리 시간** | 5분 영상 기준 다운로드~Gemini 전체 완료 60초 이내 (Flash 모드) |
| **자동 복구** | 백엔드/터널 크래시 시 launchd가 10초 내 재시작 |
| **데이터 무손실** | 영상/HTML/스크립트는 한 번 생성되면 자동 삭제 안 함 (사용자가 지우기 전까지) |
| **보안** | 백엔드는 127.0.0.1 only 바인딩. 외부 접근은 터널 + Firebase 토큰 검증을 통해서만 |
| **이식성** | macOS 12+ Apple Silicon/Intel. install.sh 한 줄로 새 맥에서 동작 |

## 6. 사용자 흐름

```
[휴대폰에서 영상 발견]
        ↓
[텔레그램 봇에 URL 전송] ────┐
        OR                  │
[웹 페이지에 URL 입력]       │
        ↓                   ↓
[잡 등록] ←─────────────────┘
        ↓
[FastAPI 백엔드: 다운로드 → 자막 → 요약]
        ↓
[로컬 디스크에 5종 산출물 저장]
        ↓
[푸시 알림 (텔레그램) + UI 업데이트]
        ↓
[PC/모바일에서 결과 확인 → 이메일 복사 / 블로그 붙여넣기]
```

## 7. MVP 범위 밖 (향후)

- Slack/Discord 봇 (텔레그램과 같은 패턴으로 추가 가능)
- Whisper로 자막 없는 영상 처리 (현재는 자막 없으면 Gemini 건너뜀)
- 잡 병렬 처리 (현재 워커 1개)
- 영상 검색/필터/태그
- 다중 사용자 (현재는 개인용 가정)
- Firestore 메타 동기화 (현재는 로컬 job.json 만)
- 영상 클립 편집 / 썸네일 생성

## 8. 성공 기준

| 지표 | 목표 |
|------|------|
| 텔레그램에 URL 보낸 후 첫 응답까지 | 5초 이내 |
| 5분 영상 전체 처리 시간 (Flash) | 60초 이내 |
| 모바일 한 손 조작 가능 화면 | 4개 (입력/목록/상세/설정) |
| install.sh 한 번 실행 시 새 맥 셋업 시간 | 5분 이내 |
| 자동 재시작 성공률 | 99%+ |
